## Checklist technique

### Avant la présentation

- [ ] Application déployée et accessible par URL
- [ ] Testé sur le réseau/wifi du lieu
- [ ] Compte de démo créé (si authentification requise)
- [ ] Données de test pré-chargées pour la démo
- [ ] Plan B préparé (vidéo ou screenshots)
- [ ] Chronométré le pitch au moins 2 fois
- [ ] Testé sur l'ordinateur de présentation si possible

---

## Points clés à retenir

### À FAIRE
- [ ] Commencer par le problème, pas la techno
- [ ] Montrer un prototype qui fonctionne
- [ ] Rester focus sur ce qui existe maintenant
- [ ] Parler avec confiance de ce qui a été accompli
- [ ] Préparer des réponses aux questions probables

### À ÉVITER
- [ ] Passer 2 min sur la stack technique
- [ ] Dire "c'est juste un prototype"
- [ ] Lancer npm/docker devant le jury
- [ ] Parler de tout ce qu'on "pourrait ajouter"
- [ ] Naviguer dans le code pendant la démo
